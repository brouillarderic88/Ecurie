# ADR-002 — Documentation en Markdown sur GitHub

**Date** : 20 mars 2026  
**Statut** : Accepté

## Contexte
Besoin de centraliser la documentation sans dépendre d'un service cloud tiers (Notion inaccessible depuis Lausanne).

## Décision
Utiliser des fichiers **Markdown** versionnés dans le **dépôt GitHub** du projet.

## Alternatives écartées
- Notion : service inaccessible au moment du choix
- AppFlowy : nécessite une installation locale
- Confluence : payant, trop lourd pour un projet solo

## Conséquences
- Documentation et code au même endroit
- Historique des modifications via Git
- Lisible directement sur GitHub sans outil supplémentaire
