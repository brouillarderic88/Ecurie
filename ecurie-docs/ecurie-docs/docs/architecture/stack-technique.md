# Stack technique

> À compléter — décision en cours

## Options envisagées

### Frontend mobile
| Option | Avantages | Inconvénients |
|---|---|---|
| React Native | Une seule codebase iOS+Android | Performances natives limitées |
| Flutter | Performances élevées | Langage Dart moins courant |
| Swift / Kotlin natif | Meilleures performances | Deux codebases à maintenir |

### Frontend web
| Option | Avantages | Inconvénients |
|---|---|---|
| React | Très répandu, large écosystème | Courbe d'apprentissage |
| Vue.js | Plus simple à démarrer | Écosystème plus petit |
| Next.js | SEO, full-stack possible | Plus complexe |

### Backend
| Option | Avantages | Inconvénients |
|---|---|---|
| Node.js + Express | Même langage que React Native | Moins structuré |
| Laravel (PHP) | Très complet, rapide à développer | PHP moins tendance |
| Django (Python) | Robuste, admin intégré | Moins adapté au temps réel |

### Hébergement
| Option | Avantages | Inconvénients |
|---|---|---|
| Render | Simple, gratuit pour démarrer | Moins puissant |
| Railway | Simple, moderne | Plus cher à l'échelle |
| AWS / GCP | Très puissant | Complexe pour débuter |
